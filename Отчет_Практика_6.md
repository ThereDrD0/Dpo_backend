# Отчет по практике 6

Тема: конфигурация ASP.NET Core, разные провайдеры, Bind/Get и IOptions.

В проекте сделан учебный центр конфигурации `CampusHub.ConfigCenter`. Старые маршруты из прошлой практики оставлены, но для этой работы добавлены маршруты `/config/*`.

Проверенные запросы:

- `/` - стартовая страница со списком маршрутов.
- `/config/raw` - вывод значений через `IConfiguration`.
- `/config/section/portal` - чтение секции `Portal` через `GetSection()`.
- `/config/tree?section=Portal` - рекурсивный вывод секции через `GetChildren()`.
- `/config/connection` - строка подключения через `GetConnectionString("DefaultConnection")`.
- `/config/providers` - реальные провайдеры из `IConfigurationRoot.Providers`.
- `/config/custom` - значения из `customsettings.txt`, прочитанные собственным провайдером.
- `/config/bind` - объект `PortalOptions`, полученный через `Bind()`.
- `/config/options` - значения из `IOptions<PortalOptions>` и `IOptions<NotificationOptions>`.
- `/config/effective` - итоговые значения конфликтующих ключей.

Порядок подключения дополнительных источников в `Program.cs`:

```csharp
builder.Configuration
    .AddXmlFile("portal.xml", optional: false, reloadOnChange: true)
    .AddIniFile("notifications.ini", optional: false, reloadOnChange: true)
    .AddTextFile("customsettings.txt")
    .AddInMemoryCollection(new Dictionary<string, string?>
    {
        ["Notifications:Sender"] = "memory-notify@college.local"
    });
```

Стандартные `appsettings.json`, `appsettings.Development.json`, переменные окружения и `commandLineArgs` подключаются ASP.NET Core автоматически при создании `WebApplicationBuilder`.

В `launchSettings.json` добавлены:

```json
"environmentVariables": {
  "ASPNETCORE_ENVIRONMENT": "Development",
  "Portal__SupportEmail": "support-env@college.local"
},
"commandLineArgs": "--Portal:Title=\"CampusHub From Command Line\""
```

Конфликты ключей:

- `Portal:Title` есть в `appsettings.json`, `appsettings.Development.json` и `commandLineArgs`. Победило значение из `commandLineArgs`: `CampusHub From Command Line`.
- `Portal:SupportEmail` есть в `appsettings.json` и в переменной окружения `Portal__SupportEmail`. Победила переменная окружения: `support-env@college.local`.
- `Notifications:Sender` есть в `notifications.ini` и в `AddInMemoryCollection()`. Победило in-memory значение: `memory-notify@college.local`, потому что оно подключено позже.

`Bind()` просто берет секцию конфигурации и заполняет обычный объект. В моем случае `/config/bind` создает `PortalOptions` и заполняет его из секции `Portal`.

`IOptions<T>` нужен, когда настройки надо зарегистрировать в контейнере зависимостей и получать их в endpoint или middleware как сервис. В проекте так зарегистрированы `PortalOptions` и `NotificationOptions`.

Собственный провайдер работает через три класса:

- `TextConfigurationProvider` читает файл.
- `TextConfigurationSource` создает провайдер.
- `TextConfigurationExtensions` добавляет метод `AddTextFile()`.

Формат `customsettings.txt` простой: одна строка - ключ, следующая строка - значение. Например:

```text
Custom:Owner
Student lazy mode
```

Проверка заголовков: запрос к `/config/raw` вернул заголовки `X-Portal-Title: CampusHub From Command Line` и `X-Portal-Semester: 2026 development`.

Проект проверен командой `dotnet build`. Сборка прошла без ошибок и предупреждений.
